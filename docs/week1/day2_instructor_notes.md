# Day 2 Instructor Notes

## How to explain metadata simply

Try this analogy:
- **Data** answers “What measurements are here?”
- **Metadata** answers “What do the measurements mean, who made them, and how can we correctly reuse them?”

Use one recurring sentence:
> “Good metadata is what allows other people to interpret your data correctly without asking you first.”

## Good analogies (quick)

- A dataset is a book; metadata is the catalog entry and the back cover description.
- Metadata is a “translation layer” between the dataset and future users.

## Typical student confusion

1. **Metadata vs data**: students mix the dataset values with the description fields.
2. **Identifier vs URL**: students think an identifier (e.g., DOI) is the same as a download link.
3. **Standard differences**: students expect all standards to contain exactly the same fields.
4. **XML vs JSON**: students assume one format is “better” rather than understanding that they are different serializations.

## Connecting metadata to FAIR

Make the mapping explicit:
- **Findable:** metadata enables discovery; identifiers (DOI) help locate the dataset.
- **Reusable:** metadata explains meaning, licensing, and context.

Practical teaching cue:
> “When students struggle to decide if a dataset is Reusable, ask: does it explain variables, provenance, and usage terms?”

## Suggested live demonstrations

If you want to demonstrate something live during class (optional, not required for the repository):
- Show an example metadata record from a real repository
- Show how a researcher finds a dataset using its metadata

In this Week 1 repository, the main parsing demonstrations are offline using provided example files.
