# In-Class Practical (Day 1): FAIR Assessment of a Public Dataset

## Title

**FAIR Quick Assessment: Findability, Accessibility, Interoperability, and Reusability**

## Objective

Students practise producing a short, FAIR-oriented assessment summary for a dataset by identifying a **persistent identifier** (preferably a DOI) and evaluating the dataset against FAIR in a beginner-friendly way.

## Estimated time

55 minutes (adjustable)

## Task description (classroom-friendly)

1. Pick a public dataset from a repository that provides persistent identifiers.
2. Identify the dataset’s persistent identifier (target: DOI).
3. Use a FAIR assessment tool to evaluate the dataset (teaching demo).
4. Write a short FAIR result summary with strengths and improvement suggestions.

## Suggested repositories (for students)

- Zenodo
- Figshare

## Suggested FAIR tools (for students)

- F-UJI
- FAIR Evaluator

## Working mode

Students may work **individually or in pairs**.

## Expected output from students (what to submit)

Students should submit a filled template containing:
- dataset title
- repository
- DOI / persistent identifier
- short FAIR result summary
- 2 strengths
- 2 weaknesses or improvement suggestions

## Very short instructor demo workflow (before students begin)

This is a lightweight demonstration idea (you can adapt it to your available internet access).

1. Choose one public dataset (e.g., from Zenodo).
2. Locate the DOI and basic dataset documentation (title, access conditions, license, and documentation).
3. Open a FAIR tool (F-UJI or FAIR Evaluator) and run a quick assessment.
4. Demonstrate how to convert tool output into short, readable strengths and improvements.

## Mock example result (for demonstration)

**Dataset title:** `2025 Global Climate Data (Sample Project)`

**Repository:** `Zenodo (example)`

**DOI / persistent identifier:** `10.0000/example-doi`

**Short FAIR result summary:**
`The dataset is Findable due to its DOI and has accessible metadata. It supports reuse through a clear license and descriptive fields, but additional documentation about units and variable definitions would improve Interoperability.`

**Strengths (2):**
1. `DOI provides strong Findability.`
2. `Clear license and dataset description support Reusability.`

**Weaknesses / improvement suggestions (2):**
1. `Add richer metadata describing measurement units and variable definitions.`
2. `Provide machine-readable formats or clear standards for interoperability.`
