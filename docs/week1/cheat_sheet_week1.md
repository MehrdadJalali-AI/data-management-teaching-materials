# Week 1 Cheat Sheet (Terms + Quick Guidance)

## Key Day 1 terms

- **Data management:** practices for storing, documenting, securing, and sharing data for future use.
- **Data types:** structured, semi-structured, unstructured.
- **OLTP vs OLAP:** transactions vs analysis/reporting.
- **FAIR:** Findable, Accessible, Interoperable, Reusable.

## Key Day 2 terms

- **Metadata:** information that describes data.
- **Metadata types:**
  - descriptive (what it is about),
  - structural (how it is organized),
  - administrative (access, licensing, management).
- **Metadata standards:**
  - **Dublin Core:** simple, general-purpose description.
  - **DataCite:** DOI-oriented citation metadata.
  - **schema.org (JSON-LD):** web discovery metadata.

## FAIR summary (short)

- **Findable:** identifiers + searchable metadata.
- **Accessible:** clear access conditions and retrievable metadata.
- **Interoperable:** common formats and clear meaning.
- **Reusable:** license + provenance + enough documentation.

## Metadata standards: quick comparison

- **Dublin Core** helps with human-friendly description.
- **DataCite** helps with DOI-style citation and dataset reference.
- **schema.org** helps with discovery and structured web indexing.

## Common Python modules used (by notebooks)

- `pathlib` for filesystem paths
- `json` for JSON-LD files
- `xml.etree.ElementTree` for XML parsing
- `pandas` for tables and simple validation scores
