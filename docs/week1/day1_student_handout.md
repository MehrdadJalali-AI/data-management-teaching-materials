# Day 1 Student Handout (Foundations and FAIR)

## Short summary

Today you will learn what **data management** means and why it matters. You will also learn the **FAIR principles** (Findable, Accessible, Interoperable, and Reusable) and apply them in a short in-class assessment activity.

## What is data management?

Data management is the practice of keeping data:
- understandable (documentation + context),
- usable (correct formats + metadata),
- safe (protection and proper access control),
- shareable for the right audience (and reusable when allowed).

## Key examples from today

### Findable
A person can locate a dataset again later using a persistent identifier (for example, a DOI) and searchable metadata.

### Accessible
The dataset (or at least its metadata and access instructions) can be retrieved under clear access conditions.

### Interoperable
The dataset uses standards or clear descriptions so it can be combined with other datasets.

### Reusable
Someone else can use the dataset correctly because licensing, provenance, and documentation are clear.

## FAIR quick reference

- **F (Findable):** DOI/identifier + rich metadata + searchable indexing.
- **A (Accessible):** retrievable via standard protocol; metadata stays available even if data access is restricted.
- **I (Interoperable):** common formats, vocabularies, and meaningful links.
- **R (Reusable):** clear license + documentation of context, methods, and data meaning.

## Reflection questions (5)

2. What is the most common reason a dataset becomes “hard to reuse”?
3. If a dataset has a DOI, does it guarantee that it is reusable? Why or why not?
4. Which FAIR principle would you prioritize first when improving a dataset?
5. What is one question you would ask before deciding to reuse a dataset?
