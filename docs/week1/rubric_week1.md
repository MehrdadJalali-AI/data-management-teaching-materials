# Week 1 Rubric

## Day 1 Practical (FAIR Assessment)

Assessment criteria (simple):
1. **Correctness:** identifier and FAIR-related claims are consistent and plausible.
2. **Completeness:** includes required fields (title, repository, DOI/identifier, FAIR summary, 2 strengths, 2 weaknesses).
3. **Clarity:** writing is readable and specific (no vague statements like “it is good”).
4. **Practical understanding:** links observations to FAIR principles (Findable/Accessible/Interoperable/Reusable).

## Day 2 In-Class Exercise (Improve Partial Dublin Core)

Assessment criteria:
1. **Correctness:** XML is valid and uses appropriate fields for Dublin Core.
2. **Completeness:** fills the template’s required elements (title, creator, publisher, identifier, description, subject, date).
3. **Clarity:** field values are meaningful and consistent with the dataset.
4. **Practical understanding:** shows understanding of how metadata supports FAIR Findability/Reusability.

## Day 2 Homework (Create DataCite + schema.org Dataset Metadata)

Assessment criteria:
1. **Correctness (structure):** XML/JSON-LD syntax is valid.
2. **Completeness (core fields):** includes core teaching fields (title, creator, identifier, description; plus additional fields if available).
3. **Clarity (meaning):** chosen fields match the real dataset.
4. **FAIR connection:** demonstrates a basic relationship between fields and Findability/Reusability.
