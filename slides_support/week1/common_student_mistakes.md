# Common Student Mistakes (Week 1)

## Likely misunderstandings about FAIR

- Treating FAIR as the same thing as “open access.”
- Assuming Findable automatically implies Reusable.
- Thinking Interoperable only means the same file extension.
- Writing a FAIR claim without referencing any evidence from the dataset documentation.

## Likely misunderstandings about metadata

- Confusing metadata (descriptive information) with the dataset’s actual data values.
- Believing all standards include identical fields.
- Mixing up an identifier (e.g., DOI) with a temporary download URL.
- Assuming a dataset’s “metadata exists somewhere,” without checking it.

## XML vs JSON confusion

- Thinking XML is always “structured” and JSON is always “unstructured.”
- Mixing up which standard uses which format (e.g., Dublin Core and DataCite are often XML; schema.org is often JSON-LD).

## Identifier vs URL confusion

- Using a URL as an identifier without checking if it is persistent.
- Overlooking that DOIs resolve via a persistent mechanism and should not be replaced with temporary links.

## Metadata vs data confusion

- Copying dataset values into metadata fields (e.g., putting rows of the CSV into `description`).
- Writing that “metadata is inside the CSV,” without describing which column names or variable definitions act as metadata.
